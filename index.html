4. Technical Implementation

4.3 Algorithm Details

4.3.1 TF-IDF Vectorization

TF-IDF(t, d) = TF(t, d) × IDF(t) where: - TF(t, d) = Term Frequency in document - IDF(t) = Inverse Document Frequency - t = term, d = document

4.3.2 Cosine Similarity

similarity = (A · B) / (||A|| × ||B||) where: - A = query vector - B = FAQ question vector - Result: value between 0 and 1

4. Technical Implementation

4.3.3 Answer Selection Algorithm

1. Transform user question to TF-IDF vector 2. Calculate cosine similarity with all FAQ questions 3. Find maximum similarity score 4. If similarity >= 0.35: - Return corresponding answer 5. Else: - Return "I don't know" message

6. Model Selection

6.1 Model Selection Criteria

  • Accuracy: High precision in answer retrieval
  • Speed: Fast response time (< 1 second)
  • Simplicity: Easy to implement and maintain
  • Interpretability: Understandable matching process
  • Resource Efficiency: Low computational requirements
  • Scalability: Handle growing FAQ database

6.2.1 TF-IDF + Cosine Similarity (Selected)

Advantages

  • Simple and interpretable
  • Fast computation
  • No training required
  • Works well with small datasets
  • Low memory footprint

Disadvantages

  • Limited semantic understanding
  • May miss synonyms
  • Requires exact word matches

6. Model Selection

6.3 Final Model Architecture

Input: User Question (text) ↓ TF-IDF Vectorization ↓ Cosine Similarity with FAQ Questions ↓ Best Match Selection (argmax) ↓ Threshold Check (>= 0.35) ↓ Output: Answer or "Don't Know" message

6.3.2 Model Parameters

  • Vectorizer: TfidfVectorizer from scikit-learn
  • Similarity Metric: Cosine Similarity
  • Threshold: 0.35 (tuned for optimal performance)
  • Normalization: Lowercase, whitespace stripping

Why Selected:

Best fit for FAQ retrieval task, sufficient accuracy for domain-specific questions, fast response times, easy maintenance and updates.

7. Model Training

7.3 Threshold Tuning

7.3.1 Similarity Threshold Selection

Selected Threshold: 0.35

Purpose: Filter out irrelevant matches

Tuning Method: Empirical testing with sample queries

High Threshold (> 0.5)

  • Higher precision
  • Lower recall
  • More "I don't know" responses

Low Threshold (< 0.3)

  • Higher recall
  • Lower precision
  • More incorrect answers

Optimal: 0.35

Balanced precision and recall - Too low returns irrelevant answers, too high rejects valid questions.
